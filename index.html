A few things I've picked up during week 1 that is worth remembering:

Why we always should name our main page index.html:

The text before .html decides which page we are going to land on and also decides the structure of the html-link.

index.html always loads the indexfile such as: http://www.examlple.com

Choosing anonther name such as about.html will land you on a different page that looks like: http://www.example.com/about.html

Shortcuts that are good to know while working in Visual Studio Code:

Command+S - Saves your changes, this is a must in order to get your text to show up on your website. (To make this automated, install "Live Server" to your Visual Studio Code application, then just right-click somewhere in your index-file and choose "Open With Live Server").

Another way to open your Live Server is by using the short-command: Command+L and then Command+O.

To stop using Live Server you use the commands: Command+L and then Command+C.

To undo the latest action, use: Command+Z.
To redo an action, press: Command+Y.

Shift+Command+P - Opens up the Command Palette.

Things I've noticed that can make future work easier:

If you highlight a tag with your cursor, all tags of the same kind will become highlighted as well, making it easier to navigate.

The esiest way to open your webpage on your browser of choice is to install "open in browser" to your Visual Studio Code application. This allows you to right-click in your index-file and choose "Open In Default Browser" or "Open In Other Browsers".
If you'd like to open your webpage via Visual Studio Code in your default browser an even quicker way after installing "open in browser", use command: alt+B.

You can easily make comments in your work by using the tag: < !-- -- >, but without the spaces. These will not show up on your actual website.
